React + Flux.

Flux é aplicação arquitetural para criar interface para usuários.
Criado pelo grupo Facebook é um complemento para a lib React criar aplicações front web.

Flux utiliza o processo de one way data flow, seguindo um fluxo unidirecional. Quando a aplicação é unidirecional ela se torna mais previsível.

Flux é composto por 3 partes: dispatcher, stores e view.

Basicamente quando uma ação é acionado na camada View, uma action é acionada e notifica a camada Dispatcher que dispara um callback para todos os Stores registrados.

  • Dispatcher é o ponto central e controla todo fluxo de dados da aplicação. Responsável somente por orquestrar o fluxo de dados, nenhuma inteligência ou regra de negócio estão nessa camada.
  • Stores é a camada onde a lógica ou regra de negócio acontece. Após toda operação concluída a camada View é atualizada.
  • View é a camada de apresentação, e gera action para a camada de Dispatcher e é atualizada pela camada de Store.

Criando um projeto com flux.

O pré requisito para utilizar o React é ter o Nodejs com a versão estável mais atual instalado.

$ sudo apt-get update
$ sudo apt-get install nodejs
$ sudo apt-get install npm

Iniciando seu projeto:

Se já tem o React instalado, pule a etapa abaixo:
$ npm install -g create-react-app

Criando um projeto "my-app":
$ create-react-app my-app

Entre no projeto "my-app" e apague o conteudo da pasta "src":

Conclusão.

O Flux não vem para substituir o consolidado MVC, mas a primeira impressão que tive desse modelo que utiliza o conceito de "One way dataflow" foi espetacular. Como o fluxo só tem uma direção o controle sobre os dados ficam mais simples e precisos, além disso os dados se tornam imutáveis, pois cada evento gera um objeto novo.
