Esse artigo vai ser bem curto, pois não existe muita diferença com modificadores de outras linguagens.
Modificador é a capacidade de alterar a visibilidade de uma classe, função ou variável.

Modificadores

Assim como em outras linguagens, em Kotlin, classes, objetos, interfaces, funções, variáveis e construtores podem ter sua visibilidade modificada.
Podem ser usados os modificadores private, public, protected e internal. Se nenhum modificador for configurado, Kotlin por padrão assume que o modificador é publico.



Private

Declarando o modificador como private , tornamos uma função ou variável seja acessível somente dentro da classe.
Uma classe não pode ser declarada como privada.

Modificador private
private var color: String // Variável visível somente dentro da classe

private metodo1 ( ) { ... } // Pode ser acessado somente dentro da classe


//Podemos declarar um construtor como private
class Teste private constructor (a: Int) { ... }



Protected

Parecido com private, o modificador protected pode ser visível também por sub classes.

Modificador Protected
open class Teste( ) {
        protected var foo: String
}


class Testando( ) : Teste {
        // a variável foo é visível.
}


class Testando( teste: Teste ) {
        // teste.foo (variável foo) não é visível.
}




Public

Declarando o modificador como public , tornamos uma variável, classe ou função visível para todos.

Modificador public
open class Teste( ) {
        public var foo: String
}


class Testando( ) : Teste {
        // a variável foo é visível.
}


class Testando( teste: Teste ) {
        // teste.foo (variável foo) é visível agora.
}




internal

Com modificador internal, tornamos a variável visível por qualquer módulo.

Modificador internal
class Teste( ) {
        internal var foo: String
}


class Testando( teste: Teste ) {
        // teste.foo (variável foo) é visível agora se estiver no mesmo módulo.
}
