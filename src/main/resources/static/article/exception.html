Checked e Unchecked Exception.

Você sabe a diferença entre uma Checked e uma Unchecked Exception em Java?

Muitos desenvolvedores Java não sabem a diferença ou não sabem quando utilizar os 2 tipos de exceptions.

De uma forma simples de explicar, uma Checked Exception é quando conseguimos prever e tratar algumas exceções.

Já uma Unchecked Exception, é quando não conseguimos prever quando o erro pode acontecer e ou não queremos tratar o erro pois ela lança automaticamente uma Exceção.

Além disso uma Unchecked Exception herda RuntimeException e não necessita de um bloco try/catch para lançar uma mensagem de exceção.

Exemplos de Checked/Unchecked Exception.

Uma Checked Exception como falamos no tópico acima é quando conseguimos prever algum problema que pode ocorrer.

Um exemplo de uma Checked Exception, é uma validação de campos, podemos tratar uma exceção caso os campos não estejam preenchidos corretamente.

Já uma Unchecked Exception é uma exceção que não prevemos, como uma falha no banco de dados ou uma conexão com a rede.

Nesse tipo de exceção (Unchecked) podemos fazer um tratamento para tornar a mensagem mais amigável ou simplesmente não fazer nada que ela vai lançar uma exceção automaticamente.

Independente qual seja a Exception, nunca abra um bloco de try/catch e não trate a exceção.

Conclusão.

Utilize as boas práticas para aplicar corretamente os tipos certos de exceptions, e a regra número 1 é sempre tratar uma exceção para não mascarar o erro.
