Python Namespaces e Scope.

Nas linguagens de programação o Namespaces é uma forma de organizar e estruturar os nomes dos nossos objetos.

Imagine fisicamente uma sala com algumas pessoas e todas elas têm o mesmo nome de João, como faríamos para chamar cada um deles?

Nesse caso teríamos que chamá-los pelo nome e sobrenome para identificar cada um, pode ser que alguns sobrenomes sejam iguais, mas as chances de isso acontecer é menor.

Em uma linguagem de programação é a mesma coisa, imagine como seria difícil criar uma variável ou método sem repetir o nome?

Para resolver esse problema existe o Namespaces, que vai separar por meio de escopo ou também por módulos, mas no momento não vamos focar nos módulos.

Namespaces e Scopes.

Em Python os Namespaces são divididos em 4 tipos: Built-In, Global, Enclosing e Local.

O Namespace tipo Built-In são nomes que fazem parte do core do Python e são carregados automaticamente:

O Namespace tipo Global são nomes que são introduzidos na raiz principal do código.

O Namespace tipo Enclosing são nomes denominadas para funções que estão no nível principal do código e os namespaces do tipo Local são nomes para variáveis ou funções dentro de uma outra função.

Mas como esses tipos de Namespaces ajudam o Python a encontrar um nome de uma função ou variável no meio de possíveis nomes iguais espalhados pelo código?

Aí que entra o Scope, onde o Python verifica o nome da variável ou função escalando níveis.

A primeira busca pelo nome será dentro do escopo Local, se não encontrar, a busca será dentro do escopo Enclosing, se não encontrar a buscá será pelo escopo Global e não encontrando a última tentativa será no escopo Built-In.

Assim o Python consegue administrar possíveis repetições de nomes das variáveis e funções.

Conclusão.

O Python também utiliza os pacotes ou módulos para auxiliar no Namespace como em outras linguagens como Java e C#, mas nesse Post mostramos como o Python consegue dentro do seu contexto diferenciar nomes e funções que podem estar espalhados pelo código.
