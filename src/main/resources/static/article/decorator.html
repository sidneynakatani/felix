Design pattern Decorator.

O Decorator é um padrão de projeto estrutural.
A motivação de utilização do padrão de projeto Decorator é incluir funcionalidades dinamicamente em tempo de execução.

O Decorator permite agregar novas capacidades ao objeto e somente ao Objeto, diferente de Herança onde adicionamos as capacidades para todos os objetos.

Podemos exemplificar a utilização do padrão Decorator em um sistema que monta diferentes tipos de pratos, com diversos ingredientes e cada item com seu respectivo preço. Ao final, o sistema deve exibir o valor do produto final conforme o prato e os ingredientes utilizados.


Como funciona o Decorator.

O pattern Decorator exibe um design onde podemos agregar novas funcionalidades dinamicamente.

Por adicionar as funcionalidades somente ao objeto, o Decorator tem uma flexibilidade maior que uma Herança.



Exemplo na prática(Java).

Vamos criar um exemplo de como implementar o padrão Decorator utilizando Java.

Criamos uma classe abstrata Dish que vai ser a base para as demais classes concretas que vamos utilizar para criar os diferentes pratos.



Criamos a classe concreta Pasta para nosso primeiro prato.



Criamos a classe concreta Soup para nosso segundo prato.



Criamos a classe DishDecorator que é a classe responsável por adicionar as novas capacidades no objeto.



Criamos a classe Onion para nosso primeiro ingrediente.



Criamos a classe Tomato para nosso segundo ingrediente.

Conclusão.

O padrão Decorator é útil quando existe a necessidade de agregar uma nova funcionalidade ao objeto em uso, sem modificar outros objetos como acontece em Herança.
