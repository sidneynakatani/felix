Design pattern Flyweight.

O Flyweight é um padrão de projeto estrutural.
Esse tipo de padrão de projeto é muito utilizado quando existe a necessidade de economizar memória.

Projetos onde exige uma grande necessidade de instanciar o mesmo objeto com propriedades diferentes é um forte candidato a utilizar esse padrão.

Muito parecido com o pattern Singleton, mas com a diferença que seu factory só vai ser instanciado no momento que realmente for usado. O padrão Singleton já vem a algum tempo sendo rotulado de anti-pattern por muitos especialistas, mas isso é um outro tema.

Um exemplo seria um projeto de um editor de texto, onde seria necessário instanciar varias vezes o objeto character. Utilizando o padrão Flyweight poderíamos economizar memória verificando se o objeto já foi instanciado e tomar a decisão de usá-lo.

Como funciona o Flyweight.

O pattern Flyweight exibe um design onde evitamos instanciar objetos repetidos.

Utilizando um factory que vai intermediar a inicialização do objeto, o número de objetos instanciados é reduzido conforme sua utilização.

Esse factory funciona da seguinte maneira:
Ao inicializar um objeto é verificado se esse objeto está na lista de objetos em memória, se não estiver é adicionado na lista e inicializado. Se o objeto já estiver na lista é recuperado sua referencia em memória e utilizado o mesmo.


Exemplo na prática(Java).

Vamos criar um exemplo de como implementar o padrão Flyweight utilizando Java.

Criamos uma classe Character que vai ser responsável por criar um objeto que exibe um carácter.



Criamos um factory para adicionar e verificar na lista se o objeto já foi instanciado e tomar a decisão de criar ou recuperar da memória o objeto.



Criamos a classe Demo para testar:

No teste executamos uma lista de caracteres "Green apple" para analisar o número de vezes que o objeto Character é inicializado ou recuperado da memória.



Executando aplicação.

Repare que o objeto Character que já foi utilizado uma vez não é instanciado novamente e sim recuperado em memória.





Conclusão.

O padrão Flyweight pode auxiliar em casos onde existe uma limitação de memória e/ou existe um uso repetitivo do mesmo objeto para a mesma tarefa.
