Unicode e Character Encodings.

Ainda vejo muitas desenvolvedores com dúvidas sobre Unicode e character encodings, muitas vezes por não compreender os fundamentos e padronização do Unicode e ASCII.

O Unicode é um padrão de representação de texto projetado com base no código ASCII com a diferença que o Unicode é capaz de interpretar quase todos os caracteres e símbolos.

O Character Encodings vai determinar dentro do Unicode qual o padrão de mapeamento dos caracteres e símbolos é utilizado, os mais conhecidos são UTF-8, UTF-16 e UTF-32.

Existem outros encodings como o latin-1(ISO/8859-1) mas vamos abordar somente a família UTF.

Lembrando que os padrões de Unicode e seus encodings são padrões para qualquer área de computação, mas aqui utilizaremos Python 3 para exemplos mais práticos.

Entendendo na prática com Python 3.

Algumas vezes quando estamos trabalhando com troca de mensagens entre sistemas acontecem divergência na recepção da mensagem e os caracteres ficam todos bagunçados.

Isso acontece devido a decodificação do encoding está diferente do encoding que foi utilizado pelo transmissor.

Basicamente não existe segredo, a única tarefa é identificar em qual encoding o transmissor utilizou e utilizar o mesmo encoding.

Outro cuidado que devemos observar é que sistemas operacionais, banco de dados e servidores de aplicação podem definir seus encodings.

Por exemplo, um Mac OS utiliza por padrão o UTF-8 e o Windows utiliza o padrão UTF-16.

Nesses casos podem ocorrer problemas na troca de mensagens entre os dois sistemas, o ideal é durante a construção de uma aplicação estabelecer um padrão de encoding para a leitura e escrita da transmissão de mensagem.

Conclusão.

Atenção quando trabalhar com troca de mensagem, observe qual encoding o sistema que está trabalhando utiliza para evitar problemas de decodificação dos caracteres.
