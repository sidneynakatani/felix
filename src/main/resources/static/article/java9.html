Novidades Java 9.

O grande desafio para Oracle é atualizar a JVM introduzindo e melhorando novas capacidades sem quebrar a compatibilidade com funcionalidades das versões anteriores. Além disso a performance foi melhorada assim como o garbage collector.

As novidades são grandes, mas vamos atentar nas principais capacidades que foram introduzidas no Java 9.

JShell

Uma das novidades introduzidas nesta versão é o JShell, um ambiente interativo para ser utilizado em um terminal de comando.
Agora podemos testar código Java diretamente no terminal sem criar classe com método main como em ambientes Javascript e Python.

Stream Reativo

Foi introduzido o Stream Reativo com a API Flow e utilizar o padrão publisher/subscriber.
Mais uma opção para quem não quer utilizar bibliotecas de terceiros como RxJava ou Project Reactor.

Melhorias no Compilador

Foram feitas melhorias no compilador, mas o carro chefe foi o Ahead-of-Time(AoT) onde podemos gerar código nativo para máquina, ainda em fase experimental mas já podemos testar essa capacidade da VM no GraalVm.
As aplicações Java estão se tornando muito grande e o Ahead-of-Time vem para suprir essa demanda, mas o diretor de Marketing Java e Tecnologia Dmitry Leskov ainda acha que essa capacidade não está madura e vai ser postergado para a próxima versão do Java 10.

Http/2

O Java 9 vem com a nova API Http/2 que substitui a antiga API HttpUrlConnection que tinha alguns problemas.

Garbage Collector

O GC Concurrent Mark Sweep(CMS) vai foi marcado como deprecated nessa versão, a intenção é substituir pelo G1 garbage collector.

Logging

Hoje existem várias implementações de logging, para evitar confusões, foi criado uma implementação padrão.
Uma nova API de log System.Logging para padronizar o provedor de mensagens.

Conclusão.

Java 9 vem com várias novidades e melhorias, algumas funcionalidades foram marcadas como deprecated e outras que já estavam marcadas nas versões anteriores foram substituídas ou removidas.
Enquanto escrevo esse material, o Java 10 já foi lançada(6 meses após o lançamento do Java 9), no próximo Post vamos falar um pouco sobre Java 10 e o motivo desse lançamento tão rápido.
