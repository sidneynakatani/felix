Arquitetura Hexagonal.

Já ouviu falar ou leu algo sobre Arquitetura Hexagonal?

Essa arquitetura mostra como podemos estruturar nosso projeto de forma organizada, protegida e escalável de uma forma simples.

Utilizando conceitos como Adapter/Port, Mediation e Application Domain essa arquitetura facilita muito trocas de variantes externas.

É importante lembrar que o DDD ou Domain Driven Design deve estar aplicado nessa arquitetura para que seja bem utilizado.

As vantagens em utilizar a Arquitetura Hexagonal são:

  • Proteção da camada de domínio da aplicação;
  • Separação de responsabilidade;
  • Facilidade de troca de componentes e fornecedores externos.

Observe que os ganhos utilizando essa arquitetura é grande, logicamente se for implementada corretamente.

Camadas de uma Arquitetura Hexagonal.

Dentro de uma Arquitetura Hexagonal teremos 3 camadas que devem ser implementadas de forma coesa.

  • Port/Adapter:são dois pontos da camada que ligam componentes externos como banco de dados, uma API, uma camada Front, camada Http, entre outros;
  • Mediation: essa camada faz a mediação entre a camada de domínio de aplicação e os componentes externos e faz as alterações necessárias para que a comunicação seja feita de uma forma contratual;
  • Application Domain:toda regra de negócio fica nessa camada, esse ponto não deve conhecer nada de outras camadas, somente o que está dentro do seu domínio.

Um pouco mais sobre Port e Adapter.

Essa camada é bem importante, não que as outras não sejam, mas por ser um ponto de entrada e saída ela deve ser bem construída.

O Port deve ser construída em forma de Interface para que seja reutilizável e que a troca dos componentes seja transparente.

O Adapter sempre será construído como implementação do Port.

Arquitetura Hexagonal em Microsserviços.

Essa modalidade de arquitetura em microsserviços deve-se prestar muita atenção quanto aos Domínios e o desacoplamento.
