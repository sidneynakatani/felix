Corrigir bugs rapidamente.

Os bugs são falhas ou erros nos softwares que geram inúmeros problemas para o sistema.

Às vezes achamos bugs mas achamos que não são tão problemáticos e deixamos para corrigir depois.

Essa prática pode gerar vários problemas para o sistema e desencadear outros problemas.

Razões para corrigir bugs rapidamente.

  1. Não corrigir bugs, mostra que a qualidade não é importante: Quando encontrarmos um bug e não corrigimos o mais rápido possível, mostramos para o cliente que a qualidade do software não é importante;
  2. Ficar horas discutindo sobre o bug é perder tempo: Quando encontrar um bug não fique perdendo tempo discutindo sobre o mesmo, faça a correção assim que encontrar o problema;
  3. Não corrigir bugs conduz a métricas falsas: As métricas do sistema são comprometidas quando existem bugs conhecidos no sistema, podendo gerar falsos positivos;
  4. Um bug pode camuflar outro: Corrigir um problema rapidamente pode ajudar a identificar mais rápido um outro bug que está sendo gerado pelo primeiro bug;
  5. Não corrigir bugs causam custos para o cliente: Custos são gerados para o cliente quando bugs não são corrigidos rapidamente.

Conclusão.

Corrija os bugs assim que encontrá-los e o mais rápido possível para que não gere outros bugs, e corrija de forma eficaz para não corrigir e gerar outros bugs.
