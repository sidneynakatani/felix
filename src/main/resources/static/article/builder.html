Design pattern Builder.

O Builder é um padrão de projeto de criação.
O padrão de projeto Builder é utilizado quando existe a necessidade de criar objetos complexos e separar sua representação, e o mesmo processo de criação possibilite criar diferentes representações.

Uma classe responsável por criar diversas representações de um tipo de objeto através de uma interface ou classe abstrata.

Um exemplo, seria um projeto para criar modelos de carros, onde uma classe fica responsável de criar as representações de modelos diferentes através do objeto recebido.


Como funciona o Builder.

O pattern Builder exibe um design onde podemos criar varias representações de um objeto.

Utilizando uma classe abstrata, podemos criar varias representações de classes concretas.

Esse modelo de negócio ajuda a encapsular a construção da representação, escondendo a implementação do cliente, fazendo o sistema mais flexível.

Criamos uma classe Pojo Car que vai ser o modelo de criação.



Criamos a classe abstrata CarBuilder, que vai ser a base de criação.



Criamos a classe implementadora Fiat.



Criamos a classe implementadora Honda.



Criamos a classe CarConstructor responsável por criar as representações a partir de um modelo.

Conclusão.

O padrão Builder pode ser utilizado quando existe a necessidade de criar representação diferentes de um mesmo modelo. Além disso o modelo de criação fica centralizado, isolando a representação.
