Design pattern Facade.

O Facade é um padrão de projeto estrutural.
A motivação para utilizar o padrão de projeto Facade é encapsular toda camada com métodos complexos, escondendo a regra de negócio do cliente em uma interface de nível maior.

Agrupando todos os métodos em uma classe centralizadora que vai expor os métodos sem exibir a parte complexa.
Uma API(Application Programming Interface) pode ser considerado uma Facade, pois a API vai exibir uma interface com os métodos escondendo a parte complexa.

Um exemplo, seria um projeto para criar diversas formas geométricas e expor somente uma interface para o cliente utilizar seus métodos sem expor a regra de negócio.


Como funciona o Facade.

O pattern Facade exibe um design onde podemos centralizar uma classe de "fachada" exibindo todos métodos sem expor sua complexidade.

Esse modelo de negócio ajuda a simplificar e unificar o acesso aos métodos com complexidade maior, fazendo com que o código fique mais organizado e limpo.
