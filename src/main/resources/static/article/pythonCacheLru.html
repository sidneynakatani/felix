Python LRU Caching.

Existem algumas formas de deixar a aplicação mais rápida e eficiente, e uma das formas é utilizando cacheamento.

Imagine que precisamos fazer uma aplicação que busca notícias na web, e a cada chamada precisamos fazer uma requisição, gastando processamento constantemente.

Podemos melhorar essa aplicação cacheando as requisições, e utilizar as requisições repetidas que foram armazenados no cache.

Neste Post vamos demonstrar a utilização de cacheamento padrão e o cacheamento utilizando o decorator @lru_cache.

O problema do cacheamento padrão é se o número de conteúdo aumentar drasticamente, vai acontecer um estouro de memória e quebrar a aplicação.

Cacheamento padrão e LRU.

A primeira forma que vamos apresentar é o cacheamento padrão utilizando dicionário.

No exemplo acima, o aplicação verifica se existe a chamada em cache, caso exista é utilizado o valor em memória, caso contrário é armazenado a chamada em cache e em seguida é feito a chamada.
O problema nessa abordagem é que se o número de chamadas armazenadas crescerem muito, vai acontecer um estouro de memória.

Vamos agora utilizar o cacheamento utilizando LRU configurado com tamanho máximo de 4 posições.

Vamos executar o código passo a passo, nesse primeiro estágio o cache está vazia.

Agora executamos a primeira chamada e o valor será armazenado na primeira posição.

hits=0 -> Número de chamadas repetidas
misses=1 -> Soma de chamadas feitas
maxsize=4 -> Tamanho máximo do cache
currsize=1 -> Tamanho atual do cache

Executamos novamente outra chamada diferente e o valor será armazenado na próxima área do cache.

Agora vamos executar as outras chamadas e os valores serão armazenados nas próximas áreas.

Observe que próxima chamada é uma requisição repetida, agora a aplicação vai recuperar do cache e colocá-lo na primeira posição.

Nessa próxima chamada, vamos fazer uma requisição com uma URL diferente, como o tamanho cache já está cheia, o valor mais antigo é substituído pelo novo valor.

Conclusão.

A utilização de cache pode ajudar muito na performance da aplicação, desde que seja aplicado a técnica correta.
