Prefira criar classes imutáveis.

As classes imutáveis são classes onde suas instancias não podem se modificadas.
Todas as informações contidas nela são fornecidas no momento que é criada e é mantida até o fim do seu ciclo de vida.
Geralmente essa modalidade de classe são mais seguras e geram menos erros.


Cinco regras para criar uma classe imutável

1- Não Forneça nenhum método que modifique o estado do objeto;

2- Assegure que a classe não possa ser estendida;

3- Torne final todos os campos;

4- Torne privado todos os campos;

5- Assegure acesso exclusivo a qualquer componente imutável.


Na prática

Na prática, para criar uma classe imutável precisamos criá-la como final para evitar que ela seja estendida.
Criando todos os campos como private e final, evitamos que esses campos sejam modificados e fica claro que não queremos modificações.
Se a classe precisar de alguma referencia de objetos mutáveis, assegure que os clientes não consigam obter referências desse objeto.

Conclusão.

Classes imutáveis são mais previsíveis e tem menos chances de dar erros pois são mais simples de se manipular.
