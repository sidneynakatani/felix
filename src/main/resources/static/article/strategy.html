Design pattern Strategy.

O padrão de Projeto Strategy é um padrão GoF(Gang of Four) comportamental.
O padrão comportamental facilita a comunicação entre os objetos, separam as responsabilidades e define a comunicação interna.
A motivação para o uso desse padrão é melhorar a manutenção do código e um desacoplamento por interface.



Aplicação.

O padrão Strategy é aplicado em casos onde existem muitas classes com caracteristicas iguais, mas a atuação é diferente em cada caso.
Um exemplo onde é bem utilizado o padrão Strategy são projetos onde são utilizados meios de pagamentos. Onde é criado uma interface de pagamento e suas classes implementadoras como Débito, Cartão de Crédito, Boleto Bancário, etc...
Todos os meios de pagamentos tem as mesmas caracteristicas de efetuar pagamento, mas cada uma tem uma forma de fazê-lo.
