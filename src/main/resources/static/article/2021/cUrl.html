cUrl.

O cUrl ou “Client Url” é uma ferramenta para consumir requisições HTTP via linha de comandos.

Geralmente distribuídas para plataforma UNIX, mas pode ser instalado e utilizado em outras plataformas como Windows.

Por não contar com uma interface gráfica, muitos desenvolvedores preferem utilizar ferramentas mais amigáveis como Postman.

Mas muitas vezes, estamos sem a possibilidade de utilizar uma interface gráfica, casos em que estamos trabalhando dentro de um terminal de um servidor.

Outra vantagem de se usar cUrl é a possibilidade de gerar scripts com rotinas programadas.

Executando principais comandos.

Quando estamos trabalhando com Rest, podemos chamar todos os verbos disponíveis como POST, PUT, GET, DELETE, PATCH. Mas vamos apresentar os principais.

Vamos começar com POST, para criar uma entidade: $ curl --data ‘payload’ -H ‘header’ -X POST

Exemplo: $ curl https://jsonplaceholder.typicode.com/posts -H "Content-Type: application/json" -X POST --data '{ "userId": 1, "id": 2, "title": "Testando", "body": "Teste" }'

Agora vamos testar o GET para consultar uma entidade: $ curl {URL}

Exemplo: $ curl https://jsonplaceholder.typicode.com/posts/1

Podemos alterar a entidade com PUT: $ curl --data ‘payload’ -H ‘header’ -X PUT

Exemplo: $ curl https://jsonplaceholder.typicode.com/posts -X PUT -H "Content-Type: application/json" --data '{ "userId": 1, "id": 2, "title": "Test", "body": "Tests" }'

Executando DELETE podemos apagar a entidade: $ curl --data ‘payload’ -H ‘header’ -X DELETE

Exemplo: $ curl https://jsonplaceholder.typicode.com/posts/1 -X DELETE -H "Content-Type: application/json"

Conclusão.

O cUrl é uma ótima ferramenta para utilizar em terminais, principalmente onde não contamos com a interface gráfica, além da possibilidade de gerar batches para automatizar tarefas.
