Kotlin, Classes, Interface e Herança.

Nesse artigo vamos abordar um pouco sobre classes, herança e interface.
Por se tratar de um conceito, vamos somente apresentar como é representado cada uma delas no Kotlin e algumas diferenças com outras linguagens Orientada a Objetos.

Classes

Em Kotlin, as classes podem ser escritas de algumas formas, que na minha opinião confunde um pouco principalmente quem está começando com programação.
Diferente do Java, em Kotlin você pode declarar os parâmetros do construtor na própria declaração da classe, fica a seu critério.

Parâmetro do construtor na declaração da Classe
class Carro (roda: Int) {
      ...
}

//Ou pode-se declarar no construtor


class Carro constructor (roda: Int) {
      ...
}
Declarando Classe sem construtor
class Teste{
      ...
}


Podemos declarar variáveis somente leitura e/ou variáveis alteráveis, utilizando os modificadores val e var.

Declarando variáveis alteráveis e somente leitura
class Usuario( var nome: String, val senha: String){
      ...
}

//A variável nome pode ser alterada

//A variável senha não pode ser alterada



Classes com construtores secundários:

O construtor secundário deve ser declarado com a chave constructor .

Declarando Classe com construtor secundário
class Carro ( var nome: String) {
        constructor ( nome: String, rodas: Int): this (nome){
...
}

//Dois construtores (1 e 2 parâmetros)



Instanciar uma Classe:

Diferente de outras linguagens de programação como Java, ao criar uma instância de uma Classe não precisamos declarar a palavra chave "new".

Criando uma instância de uma Classe
val teste = Teste( )
val carro = Carro("Celta")


Data Classes:

Data Class é uma classe similar ao POJO(Plain Old Java Objects) do Java, mas bem mais expressiva, escrevendo poucas linhas de código.

Criando uma data Classe
data class Usuario (val nome: String, val idade: Int)
Criando e usando uma instância de uma data Classe
val usuario = Usuario("Sidney", 38)
println("Nome: " + usuario.nome)
//Nome: Sidney


Classes Enum:

A classe do tipo Enum pode ser declarado de algumas formas:

Formas de escrever uma classe Enum
enum class Carro {
        CELTA, CORSA, OMEGA
}

//ou assim:

enum class Cor (val codigo: Int) {
        AZUL(0),
        VERDE(1),
        VERMELHO(2)
}


Herança

Em Kotlin, por padrão, todas as classes herdam a classe Any. A classe Any não é a classe java.lang.Object.
A super classe sempre deve conter o modificador open

Exemplo de herança em Kotlin
//Super classe Animal
open class Animal(nome: String)


//Classe Gato herdando Animal
class Gato(nome: String) : Animal(nome)


Classes Abstrata:

Uma classe abstrata é uma classe que não pode ser instanciada, somente herdada.

Exemplo de uma classe abstrata
abstract class Funcionario {
        fun trabalha()
}


Interfaces

A Interface em Kotlin é um pouco diferente de outras linguagens como C# ou Java. As diferenças são que em kotlin a Interface pode ter implementação(a partir do Java 7 tem essa feature utilizando na assinatura o modificador "default") e a classe implementadora não é obrigado a declarar os métodos da Interface.
Podemos implementar quantas interfaces quisermos.

Exemplo de uma Interface e sua implementação
interface Funcionario {
        fun getSalario()
        fun getTrabalhar(){
               print("Trabalhando...")
        }
}

//Implementação:

class Gerente : Funcionario {
        override fun getSalario() {
               print("R$ 1.000.000,00");
        }
}
