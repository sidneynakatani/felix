Graalvm.

Graalvm é uma nova máquina virtual desenvolvido pela Oracle. Recém lançada (versão 1.0.0), o Graalvm é uma Vm multi linguagem que nesse primeiro release compila linguagens como Java, Kotlin, C, C++, Scala, Javascript e Nodejs. Existe ainda a possibilidade de adicionar Python, Ruby e R, mas acredito que no próximo release essas linguagens já venham embarcadas.

Vantagens em utilizar Graalvm:

  • Performance : Comparado com a JVM o Graalvm é otimizado para melhorar a performance de processamento;
  • Multi linguagem : Pode-se escolher a melhor linguagem para determinadas tarefas executando no mesmo ambiente;
  • Interoperáveis : aplicações de linguagens diferentes podem se comunicar utilizando a biblioteca "polyglot";
  • Embarcabilidade : Linguagem dinamicamente embarcadas e códigos nativos com a capacidade de sandboxing;
  • Ferramentas : Baseado na JVM, o Graalvm beneficia de ferramentas como debugger e profile.

As features mais interessantes na minha opinião foi a utilização da biblioteca polyglot para combinar linguagens diferentes no mesmo código e a possibilidade de gerar executável nativo na máquina utilizando qualquer linguagem.

Misturar várias linguagens diferentes no mesmo código pode piorar o entendimento e virar uma “salada de frutas”. O propósito de mixar linguagens diferentes é para casos onde precisamos utilizar uma funcionalidade de uma biblioteca que não está disponível para a linguagem que estamos utilizando.
Outra possibilidade é quando utilizamos uma linguagem que tem a maioria das qualidades que precisamos, mas é muito lenta. Nesse caso podemos utilizar outra linguagem mais rápida e embarcar o código da linguagem que queremos utilizar.

Também podemos programar em uma linguagem favorita e gerar um executável nativo para máquina em uma simples linha de comando native-image.
A vantagem nesse processo é a performance da aplicação que fica muito melhor, em contrapartida o arquivo fica bem maior.

Para baixar o GraalVm acesse o link da Oracle.

Instalação.

Baixe o GraalVm e descompacte o arquivo em um diretório de sua preferência.

No Linux adicione o caminho do diretório bin digitando o comando:
export PATH=/caminho/do/graalvm/bin:$PATH

No Mac:
export PATH=/caminho/do/graalvm/Contents/Home/bin:$PATH

Verifique a versão:
java -version

Compile:

Agora crie um executável a partir do código Java compilado:
native-image Teste

Execute o compilado Java:
java Teste

Execute o compilado nativo:
./teste

O compilado Java levou 30 milissegundos para executar contra 5 milissegundos do compilado nativo, em contrapartida o compilado Java tem o tamanho de 500 bytes contra 5 megabytes do compilado nativo.

Conclusão.

Recém lançado, o Graalvm vem para melhorar e otimizar recursos de programação. Existem outros recursos a serem explorados e provavelmente virão nos próximos lançamentos.
