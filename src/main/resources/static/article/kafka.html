Apache Kafka.

Apache Kafka é uma plataforma de streaming distribuída.

Hoje em dia as aplicações precisam ser cada vez mais rápidas, e para isso sempre precisamos de novas soluções como API’s para facilitar e distribuir as responsabilidades.

O Kafka tem uma proposta de solução para aplicações que precisam de respostas mais rápidas e com contingência.

Imagine aplicações que precisam gerar arquivos e enviar esse anexo via e-mail, um tempo muito alto para que a aplicação faça todo esse fluxo e gere uma resposta para o cliente. Com Kafka podemos dividir a aplicação em 2, diminuindo o esforço e tempo da aplicação.

São 3 capacidades que a plataforma disponibiliza:
  1. Capacidade de Publish e subscribe, similar a filas de mensagens;
  2. Armazenar registros de Stream com fault-tolerant de forma durável;
  3. Processar os registros de Stream enquanto eles ocorrem.
As principais API’s:
  • Producer API, permite a aplicação publicar registros de Stream para 1 ou mais tópicos no Kafka;
  • Consumer API, permite a aplicação consumir registros de Stream de 1 ou mais tópicos no Kafka;
  • Streams API, permite a aplicação publicar e consumir registros de Stream de 1 ou mais tópicos no Kafka;
  • Connector API, permite que o Kafka observe alterações em aplicações ou banco de dados relacionais.
Alguns casos de uso para o Kafka.
  • Mensagens: O Kafka substitui outros Broker’s de mensagens com a vantagem de ter uma taxa de transferência mais rápida, além de contar com replicação e fault-tolerance;
  • Métricas: O Kafka pode ser utilizado para sistemas de métricas e monitoração de dados, agregado a um sistema ou aplicação, pode-se capturar dados em tempo de execução sem interromper o fluxo do sistema ou aplicação;
  • Log: O Kafka pode ser utilizado para substituir ou agregar no sistema de log. Hoje os os log’s são um grande problema para as aplicações por consumir muito I/O, utilizando Kafka para controlar log’s podemos direcionar os log’s para outros pontos sem onerar a aplicação.

Tutorial.

Esse tutorial foi feito sobre a plataforma Unix, caso esteja utilizando uma plataforma Windows, troque bin/ por bin\windows\ e nos scripts adicione a extensão .bat.

Passo 1: Baixe a ultima versão estável do Kafka 1.1.0.

Passo 2: Descompacte o arquivo e entre no diretório;

Passo 3: Inicie o Zookeeper, caso não tenha instalado, utilize o Zookeeper que vem embarcado no Kafka:
> bin/zookeeper-server-start.sh config/zookeeper.properties

Passo 4: Inicie o Kafka:
> bin/kafka-server-start.sh config/server.properties

Passo 5: Criar um tópico com 1 partição:
> bin/kafka-topics.sh --create --zookeeper localhost:2181 --replication-factor 1 --partitions 1
--topic test

Passo 6: Publicar uma mensagem:
> bin/kafka-console-producer.sh --broker-list localhost:9092 --topic test
> Oi

Passo 7: Criar um Consumidor:
> bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic test –from-beginning
> Oi

Conclusão.

O Apache Kafka vem para agregar aplicações trazendo soluções para algumas aplicações que precisam de performance e velocidade nas respostas para seus clientes. Podemos utilizar Kafka para várias soluções, desde a mais simples como transmissões de mensagens até as mais complexas como interligar várias aplicações em cluster.
