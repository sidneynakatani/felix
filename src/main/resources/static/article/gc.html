Garbage Collector.

O Java Garbage Collector é o processo automático de gerenciamento de memória.

Programas Java são executados na JVM, os objetos são criados no heap, que é uma parte da memória dedicada para aplicação.

Quando os objetos não são mais necessários, o garbage collector localiza esses objetos não utilizados e os exclui para liberar espaço em memória.

O maior benefício do Garbage Collector é o gerenciamento automático de exclusão de objetos não utilizados.

Um problema no GC é que não há como prever quando o garbage Collector ocorrerá em tempo de execução.

Tipos de garbage Collector.

O JDK oferece algumas opções de Garbage Collector:

  • Serial Collector: É o mais simples das GC e os eventos são conduzidos em uma única thread, para ativar a GC, deve-se adicionar nos argumentos da JVM: -XX:+UseSerialGC;
  • Parallel / Throughput collector: É o GC padrão, Várias threads são usadas para coleta de lixo menor e uma única thread é utilizada para coleta de lixo principal e compactação;
  • G1 Collector: O garbage collector mais recente destina-se a ser um substituto para o CMS, o G1 Collector utiliza várias threads dividindo a heap em regiões, para ligar esse modelo de GC deve-se adicionar nos argumentos da JVM: –XX:+UseG1GC.

Conclusão.

Para programadores mais avançados que querem melhorar a performance da aplicação, uma boa configuração do Garbage Collector pode ser uma ótima opção.
