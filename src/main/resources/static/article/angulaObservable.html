Entendendo Observable com Angular.

Acredito que a maior dificuldade quando começamos a trabalhar com Angular(versão maior de 2) é utilizar o Observable.

Biblioteca do pacote RX/js, o Observable tem uma curva de aprendizado um pouco mais lenta devido a uma impressão de complexidade, mas estudando um pouco mais, fica totalmente tranquila de trabalhar.

Após entender como funciona a biblioteca, conseguimos fazer coisas bem legais com uma programação assíncrona.

Programação reativa e Observable trabalham sempre juntos, pois a grande massa de informação para processar e a necessidade da informação processado na hora é cada vez maior.

Observable.

Primeiro precisamos entender que o Observable oferece técnicas de programação e comportamento assíncrona.

Então enquanto estiver estudando Rxjs esqueça programação síncrona.

Outra coisa que devemos ter em mente: Java ≠ JavaScript ≠ Typescript.

Antes do Observable, a biblioteca utilizada era o Promise, basicamente os mesmos comportamentos, mas como tudo evolui, o Observable tem mais funções que seu antecessor.

Observable utiliza o padrão Observer, onde somente quem se inscreveu no método recebe a notificação.

Outro item importante sobre o Observable é o poder de transformação utilizando o método map. Essa ferramenta é muito importante pois podemos transformar as respostas recebidas em saídas de outro tipo, o exemplo mais utilizado é entrada tipo JSON para um objeto tratado.
