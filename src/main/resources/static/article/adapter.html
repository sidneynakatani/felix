Design pattern Adapter.

O Adapter é um padrão de projeto estrutural.
A motivação de utilização do padrão de projeto Adapter é utilizar uma única interface de entrada para acessar outros tipos implementações, desde que o resultado final seja o mesmo esperado.

Imagine um sistema legado que faça uma tarefa que receba como parâmetros X, Y, Z. Agora imagine que precisamos adaptar uma nova API que faça a mesma tarefa mais rápido, mas não podemos alterar a interface legado de entrada e para ajudar, a nova API recebe como parâmetros Y, X, Z.
Perceba que apesar do resultado final ser o mesmo do sistema legado, os parâmetros de entrada estão em ordem diferente. Mesmo que tivéssemos o código fonte dessa API, não seria uma boa prática alterar o código da API para uma nova implementação.
Nesse momento é que entra a implementação de um Adapter.

Podemos exemplificar a utilização do Adapter em um sistema que gera um retângulo a partir de 2 coordenadas: Altura e Largura. Nesse mesmo sistema legado, precisamos adaptar uma nova API que vai fazer a mesma tarefa, mas os parâmetros de entrada estão invertidos.


Como funciona o Adapter.

O pattern Adapter exibe um design onde podemos utilizar classes com interfaces incompatíveis adaptando essa interface para um resultado que o cliente espera.

Esse padrão funciona muito bem quando não podemos alterar a interface do sistema legado com implementações novas.



Exemplo na prática(Java).

Vamos criar um exemplo de como implementar o padrão Adapter utilizando Java.

Criamos uma Interface Retangle.



Criamos a classe implementadora RetangleImpl, que é nossa classe legado.



Criamos a classe NewRetangle, que vai ser a nova API.



Criamos a classe RetangleAdapter, que vai ser nossa classe adaptadora.

Nesse teste executamos as duas versões de Retangle(Legado e Nova API) passando os parâmetros em ordens diferentes. Depois utilizando a nova API utilizando o adapter sem alterar a interface de entrada.

Conclusão.

O padrão Adapter pode ser utilizado quando existe a necessidade de não alterar a interface de entrada adaptando a interface da nova capacidade ou API com a já existente.
