A importância do Log.

Um dos maiores benefícios de se utilizar uma ferramenta de log na aplicação é a rastreabilidade.

Para ter uma idéia de quanto é importante ter um log na aplicação, trabalhar sem uma ferramenta de log é como andar no completo escuro, sem saber o que está acontecendo em sua volta quando os problemas acontecem.

As coisas pioram quando a aplicação vai para o ambiente de produção e problemas que não aconteciam em homologação começam a aparecer.

Desespero, estresse e aquela frase padrão: “Na minha máquina funciona”.

O log não vai evitar que isso aconteça, mas vai ajudar muito a entender o que está acontecendo e facilitar a correção.

O que logar?

Existem alguns níveis de log que precisamos entender antes de saber o que logar:

  • Info / Debug: Utilizado para apresentar algumas informações ou orientação do que está ocorrendo na aplicação, geralmente utilizado em fase de desenvolvimento;
  • Warning: Apresenta algo para se atentar, mas nada crítico;
  • Error / Critical: Erros e problemas críticos, são informações que devem estar sempre ligadas independente do ambiente que esteja a aplicação, sempre que ocorre uma exception essa informação deve ser apresentada.

Quando estamos lançando uma release para produção, principalmente quando é a primeira, é importante logar tudo que achar importante e ir removendo conforme a aplicação vai amadurecendo.

Logar muita informação gera custo e processamento, por isso conforme a aplicação amadurece podemos removendo informações que não são mais relevantes.

Também podemos fazer um log inteligente, onde podemos alterar os níveis de log que devem ser apresentados em tempo de execução.

De qualquer forma, os níveis de error / critical(Essa nomenclatura pode mudar de linguagem para linguagem) devem sempre estar ligados.

Logging com Python.

Hoje em dia já não há mais desculpas para criação de um arquivo de log, com várias ferramentas prontas, está bem fácil a implementação.

No Python existe uma ferramenta nativa chamada “logging”, que é simples e rápida de utilizar.

logging.basicConfig(filename='exampleLog.log', level=logging.DEBUG, format='%(asctime)s %(levelname)s %(funcName)s => %(message)s')

Onde “filename” é onde vamos gravar o arquivo de log, “level” é o nível de log e o “format” determinamos como será formatado o log, nesse caso a data em ordem cronológica, o nível de log que gerou o evento, nome da função e a mensagem.

Conclusão.

O log é muito importante para aplicação, mas saiba quando e onde utilizar, se por um lado é bom saber tudo que está acontecendo no sistema, por outro isso gera um custo de processamento, espaço e claro financeiro.
